Detailed Description

template<class T>
class dlaf::common::Pipeline< T >

Pipeline takes ownership of a given object and manages the access to this resource by serializing calls. Anyone that requires access to the underlying resource will get a sender, which is the way to register to the queue. All requests are serialized and served in the same order they arrive. On destruction it does not wait for the queued requests for the resource and exits immediately.

The mechanism for auto-releasing the resource and passing it to the next user works thanks to the internal wrapper object. The wrapper contains the real resource, and it will do what is needed to unlock the next user as soon as the wrapper is destroyed.
